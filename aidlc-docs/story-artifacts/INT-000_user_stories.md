## US-001: タスク管理（CRUD + 完了チェック）

**As a** チームメンバーとして、
**I want** タスクを作成・編集・削除・完了チェックしたい、
**So that** 自分や チームのやるべきことを一元管理できる。

詳細: [INT-000_US-001.md](./INT-000_US-001.md)

---

## US-002: プロジェクト管理

**As a** プロジェクトリーダーとして、
**I want** プロジェクトを作成してタスクを紐付けたい、
**So that** プロジェクトごとに進捗を管理できる。

### Acceptance Criteria
- [ ] プロジェクトを作成できる（名前・説明）
- [ ] プロジェクト一覧が表示できる
- [ ] タスク作成・編集時にプロジェクトを選択できる
- [ ] プロジェクトでタスクをフィルタリング表示できる
- [ ] プロジェクトを削除できる

---

## US-003: 締め切り日・優先度の設定

**As a** チームメンバーとして、
**I want** タスクに締め切り日と優先度を設定したい、
**So that** 何を先にやるべきか判断できる。

### Acceptance Criteria
- [ ] タスク作成・編集時に締め切り日（日付）を設定できる
- [ ] タスク作成・編集時に優先度（高/中/低）を設定できる
- [ ] タスク一覧を優先度・締め切り日でソートできる
- [ ] 締め切り日が近いタスクを視覚的に確認できる

---

## US-004: カンバンボード表示

**As a** チームメンバーとして、
**I want** タスクをカンバンボード（To Do / In Progress / Done）で見たい、
**So that** チームの作業状況を一目で把握できる。

### Acceptance Criteria
- [ ] `/tasks/board` にカンバンボードが表示される
- [ ] タスクがステータスごとのカラムに表示される
- [ ] 各カードにタイトル・優先度・締め切り日が表示される
- [ ] ボタン操作でタスクのステータスを変更できる（PoCでのドラッグ&ドロップは任意）

---

## US-005: 備品QR貸し出し/返却管理

**As a** 備品管理担当者として、
**I want** 備品にQRコードを付けて貸し出し・返却を記録したい、
**So that** 誰がどの備品を借りているか常に把握できる。

### Acceptance Criteria
- [ ] 備品を登録できる（名前・型番・説明）
- [ ] 備品一覧が表示される（貸出中/返却済のステータス付き）
- [ ] 各備品のQRコードを生成・表示できる
- [ ] QRコードからアクセスすると貸し出し/返却の操作ができる
- [ ] 貸し出し時に借用者名が記録される
- [ ] 返却操作で返却日時が記録される
- [ ] 備品ごとの貸し出し履歴を閲覧できる

# INT-000 US-001: タスク管理（CRUD + 完了チェック）

## As a
チームメンバー（プロジェクトに参加している個人）

## I want
タスクを作成・編集・削除・完了チェックできる画面が欲しい

## So that
自分やチームのやるべきことを一元管理し、何が完了済みかを把握できる

---

## Acceptance Criteria

### タスク作成
- [ ] 「新しいタスク」ボタンからタスク作成フォームを開ける
- [ ] タイトル（必須）・説明・締め切り日・優先度・プロジェクト・ステータスを入力できる
- [ ] タイトルが空のまま保存しようとするとバリデーションエラーが表示される
- [ ] 保存後、タスク一覧に新しいタスクが表示される

### タスク一覧表示
- [ ] `/tasks` にアクセスするとタスク一覧が表示される
- [ ] 各タスクにタイトル・優先度・締め切り日・ステータス・完了チェックが表示される
- [ ] 未完了タスクと完了タスクが区別して表示される（または完了タスクを非表示にするトグルがある）

### タスク編集
- [ ] タスクカードの編集ボタンから編集フォームを開ける
- [ ] 全フィールドを変更・保存できる
- [ ] 保存後、一覧の表示が即座に更新される

### タスク削除
- [ ] タスクカードの削除ボタンでタスクを削除できる
- [ ] 削除前に確認ダイアログが表示される（任意、PoC優先）
- [ ] 削除後、一覧からタスクが消える

### 完了チェック
- [ ] タスクのチェックボックスをクリックすると完了/未完了を切り替えられる
- [ ] 完了したタスクはステータスが「Done」に自動変更される（または完了フラグのみ管理）
- [ ] 完了チェックの変更はDB即時反映される

---

## 画面仕様

### `/tasks` - タスク一覧画面
- ヘッダー: 「タスク管理」タイトル + 「新しいタスク」ボタン
- フィルター: プロジェクト選択・ステータス選択・優先度選択（シンプルなセレクトボックス）
- タスクリスト: shadcn/ui の Card コンポーネントで各タスクを表示
  - タイトル（太字）
  - 優先度バッジ（高=赤、中=黄、低=緑）
  - 締め切り日
  - ステータスバッジ
  - 完了チェックボックス
  - 編集ボタン・削除ボタン

### タスク作成/編集フォーム（モーダルまたは別ページ）
- タイトル（Input、必須）
- 説明（Textarea、任意）
- プロジェクト（Select）
- 優先度（Select: 高/中/低）
- ステータス（Select: To Do / In Progress / Done）
- 締め切り日（Input type="date"）
- 保存ボタン・キャンセルボタン

## データモデル（概要）

```
tasks テーブル
- id: uuid (PK)
- title: text (NOT NULL)
- description: text
- status: enum('todo', 'in_progress', 'done') DEFAULT 'todo'
- priority: enum('high', 'medium', 'low') DEFAULT 'medium'
- due_date: date
- project_id: uuid (FK -> projects.id, nullable)
- completed: boolean DEFAULT false
- created_at: timestamp
- updated_at: timestamp
```
